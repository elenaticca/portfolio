Functional Requirements [FX]

The template provides three functional requirement blocks (REQ-F001 to REQ-F003), each using the structure below.

REQ-F001[Requirement name] [DES]
PR name
Short label identifying the requirement. Match the PR identifier.
Directive
One sentence describing what the system must do. Start with a verb — Add, Allow, Introduce, Define.
Priority
MUST
Rationales
Explain why this requirement exists. Link to a business objective, user need, or product decision.
Constraints
Detailed rules, conditions, and boundaries the system must respect. Include UI behaviours, validation rules, edge cases, state transitions, and role-based differences.
Personas
List the user roles directly impacted by or interacting with this requirement.
Use cases
Reference the use case or main flow this requirement belongs to. Can be a link, a short description, or a Gherkin scenario.
REQ-F002[Requirement name] [DES]
Priority
MAY
Structure
Same seven fields as REQ-F001.
REQ-F003[Requirement name]
Priority
SHOULD
Structure
Same seven fields as REQ-F001.
